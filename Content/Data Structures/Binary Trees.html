Binary Trees

Overview

A binary tree is a tree in which each node has at most two children. The children are usually called the left child and the right child. This left/right distinction is part of the structure: swapping the two children of a node generally produces a different binary tree.

A plain binary tree does not require any ordering rule among the values. It is a structural idea. More specialized trees add extra rules: a binary search tree orders values, a binary heap requires a completeness rule and a heap-order rule, and AVL/red-black trees add balancing rules.

Terminology

  • Root: the top node of the tree.
  • Parent and child: a node is the parent of its left and right children.
  • Leaf: a node with no children.
  • Subtree: a node together with all of its descendants.
  • Depth: the number of edges from the root to a node.
  • Height: the number of edges on the longest downward path from a node to a leaf.
  • Full node: a node with exactly two children.

In many algorithms, a binary tree is defined recursively: a tree is either empty or consists of a root, a left subtree, and a right subtree. That recursive definition is why tree algorithms often have compact recursive descriptions.

Common Shapes

  • Perfect binary tree: every internal node has two children and all leaves are at the same depth.
  • Complete binary tree: every level is full except possibly the last, and the last level is filled from left to right.
  • Balanced binary tree: the height stays proportional to \(\log n\), usually because the tree enforces a balancing rule.
  • Degenerate tree: each internal node has only one child, so the tree behaves like a linked list.

Shape matters because height controls the cost of many tree operations. A tree with \(n\) nodes can have height as small as \(\Theta(\log n)\) or as large as \(\Theta(n)\).

Traversals

A traversal visits every node in a systematic order. The most common depth-first traversals are:

  • Preorder: visit the node, then the left subtree, then the right subtree.
  • Inorder: visit the left subtree, then the node, then the right subtree.
  • Postorder: visit the left subtree, then the right subtree, then the node.

Level-order traversal visits nodes by depth, from left to right within each level. It is the traversal used by breadth-first search on a tree.

All four traversals visit each node once, so their running time is \(O(n)\). Their auxiliary space depends on the method used: recursive depth-first traversals use call-stack space proportional to the tree height, while level-order traversal uses a queue that can hold many nodes from one level.

Representations

Node and References

The most direct representation stores one object per node. Each node stores a value and references to its left and right children. 

Node:
    value
    left
    right

This representation is flexible: the tree can have any shape, and rotations or subtree moves are local reference updates.

Implicit Array Layout

Complete binary trees are often stored in arrays. For zero-based indexing, the children of index \(i\) are \(2i + 1\) and \(2i + 2\), and the parent is \(\lfloor(i - 1)/2\rfloor\). Binary heaps use this representation because their complete-tree shape makes it efficient.

A general binary tree can also be stored in an array, but arbitrary missing children can waste space or require extra markers. For general tree algorithms, node references are usually easier to reason about.

Connections

  • Expression trees: operators are internal nodes and operands are leaves.
  • Decision trees: internal nodes represent choices and leaves represent outcomes.
  • Binary search trees: add an ordering invariant that supports search.
  • Heaps: add shape and priority rules for efficient priority queues.
  • AVL and red-black trees: add balancing rules to keep search-tree height logarithmic.
