Recursion Trees

Overview

A recursion tree turns a recurrence into a picture of the recursive calls. Each node represents one subproblem, and the label on the node represents the non-recursive work done by that call.

The main move is to add the work across each level of the tree, then add the levels. This is often the clearest way to understand divide-and-conquer running times.

Anatomy of a Recursion Tree

Consider a recurrence of the form

\[ T(n)=aT(n/b)+f(n). \]

The tree has one root for the original problem of size \(n\). Each internal node creates \(a\) children, each with input size divided by \(b\). The work written inside a node is the local work \(f(\text{that node's input size})\).

Tree feature Meaning in the recurrence
Root The original call on input size \(n\).
Children The recursive calls made by one call.
Node label The non-recursive work done by that call.
Level sum Total non-recursive work done by all calls at the same depth.
Leaves Base-case calls.

Example: Merge Sort

Merge sort has the recurrence

\[ T(n)=2T(n/2)+n,\quad T(1)=1. \]

The root does \(n\) merge work. The next level has two subproblems of size \(n/2\), each doing \(n/2\) local work, for a level total of \(n\). The next level has four subproblems of size \(n/4\), again for a level total of \(n\).

Level Number of subproblems Size of each subproblem Work per subproblem Total level work
0 \(1\) \(n\) \(n\) \(n\)
1 \(2\) \(n/2\) \(n/2\) \(n\)
2 \(4\) \(n/4\) \(n/4\) \(n\)
\(i\) \(2^i\) \(n/2^i\) \(n/2^i\) \(n\)

The recursion reaches size 1 when \(n/2^i=1\), so there are \(\log_2 n\) splitting levels, plus the leaf level. Since each level costs \(\Theta(n)\), the total work is \(\Theta(n\log n)\).

Example: Binary Search

Binary search has only one recursive call per non-base case:

\[ T(n)=T(n/2)+1. \]

Its recursion tree is a single path, not a branching tree. Each level has one node and constant work.

\[ 1 + 1 + 1 + \cdots + 1. \]

The input size halves until it reaches 1, so the path has \(\Theta(\log n)\) nodes. The total work is \(\Theta(\log n)\).

When Leaves Dominate

In some recurrences, the total work grows as we move down the tree. For example:

\[ T(n)=4T(n/2)+n. \]

The level costs are

\[ n,\quad 4(n/2)=2n,\quad 16(n/4)=4n,\quad \ldots \]

The work doubles each level. By the time the tree reaches the leaves, there are \(4^{\log_2 n}=n^2\) leaf calls. The bottom levels dominate, so \(T(n)=\Theta(n^2)\).

When the Root Dominates

Other recurrences spend most of their work near the top:

\[ T(n)=2T(n/2)+n^2. \]

The level costs are

\[ n^2,\quad 2(n/2)^2=\frac{n^2}{2},\quad 4(n/4)^2=\frac{n^2}{4},\quad \ldots \]

Each level costs half as much as the level above it. The total is a geometric series bounded by a constant multiple of \(n^2\), so \(T(n)=\Theta(n^2)\).

Level-Sum Template

For \(T(n)=aT(n/b)+f(n)\), the level-\(i\) information usually looks like this:

Quantity Value at level \(i\)
Number of nodes \(a^i\)
Input size per node \(n/b^i\)
Work per node \(f(n/b^i)\)
Total work on level \(a^i f(n/b^i)\)
Depth to size 1 \(\log_b n\)

The full running time is the sum of the level costs plus the base-case work at the leaves.

Connection to the Master Theorem

The Master Theorem is a shortcut for the same comparison a recursion tree reveals. It asks whether the work per level grows, stays roughly equal, or shrinks as the tree goes downward.

  • Root-dominated: Level costs shrink, so the top-level work controls the answer.
  • Balanced across levels: Each level costs about the same, so multiply one level by the number of levels.
  • Leaf-dominated: Level costs grow, so the bottom of the tree controls the answer.

Limitations

Recursion trees are excellent for insight, but they can become messy when the recursive calls have different sizes, when floors and ceilings matter, or when the recurrence branches in several unrelated ways. In those cases, the tree may still suggest a good guess, but the substitution method is often used to prove the bound carefully.

Reading Comprehension Questions

  1. What does one node in a recursion tree represent?
  2. Why does merge sort have \(\Theta(n)\) work on each level of its recursion tree?
  3. What does it mean for a recurrence to be leaf-dominated?
  4. How does a recursion tree help explain the Master Theorem?
  1. Answer: It represents one recursive call or subproblem, labeled with the non-recursive work done by that call.
  2. Answer: At level \(i\), there are \(2^i\) subproblems of size \(n/2^i\), so the total merge work is \(2^i(n/2^i)=n\).
  3. Answer: The work grows as the tree goes downward, so the bottom levels or leaves determine the asymptotic total.
  4. Answer: It shows whether level costs shrink, stay balanced, or grow, which is exactly the comparison the theorem summarizes.

In-Class Activities

  1. Draw levels: Draw the first four levels for \(T(n)=3T(n/2)+n\) and write the level totals.
  2. Classify: Decide whether \(T(n)=2T(n/2)+n^2\), \(T(n)=2T(n/2)+n\), and \(T(n)=4T(n/2)+n\) are root-dominated, balanced, or leaf-dominated.
  3. Tree to formula: Convert a recursion tree for \(T(n)=T(n/2)+1\) into a summation.

Homework Problems

  1. Balanced tree: Use a recursion tree to solve \(T(n)=3T(n/3)+n\).
  2. Leaf growth: Use a recursion tree to give a tight bound for \(T(n)=5T(n/2)+n\).
  3. Root growth: Use a recursion tree to give a tight bound for \(T(n)=2T(n/2)+n^3\).
  4. Explain in words: For each previous recurrence, explain which levels dominate and why.
