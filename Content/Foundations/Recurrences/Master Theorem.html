Master Theorem

Overview

The Master Theorem is a shortcut for many divide-and-conquer recurrences. Instead of expanding the recurrence by hand, we identify the number of subproblems, the shrink factor, and the non-recursive work, then compare how much work appears across levels of the recursion tree.

The version on this page is the common polynomial-work version used in many first algorithm analyses. More general versions exist, but this one is enough for many standard examples.

Required Form

The recurrence must fit this shape:

\[ T(n)=aT(n/b)+f(n), \]

where \(a\geq 1\), \(b>1\), and \(f(n)\) is the non-recursive work done by the current call. In the simplified version here, we assume \(f(n)=\Theta(n^d)\) for some \(d\geq 0\).

Symbol Meaning
\(a\) Number of recursive calls made by one non-base call.
\(b\) Shrink factor. Each subproblem has size \(n/b\).
\(f(n)\) Work done outside the recursive calls.
\(d\) The exponent in the polynomial local work \(f(n)=\Theta(n^d)\).

The Theorem

If

\[ T(n)=aT(n/b)+\Theta(n^d), \]

then compare \(a\), the number of subproblems, with \(b^d\), the amount by which polynomial work shrinks from one level to the next.

Comparison Tree behavior Result
\(a<b^d\) Level costs shrink; top work dominates. \(T(n)=\Theta(n^d)\)
\(a=b^d\) Each level costs about the same. \(T(n)=\Theta(n^d\log n)\)
\(a>b^d\) Level costs grow; leaves dominate. \(T(n)=\Theta(n^{\log_b a})\)

How to Use It

  1. Rewrite the recurrence in the form \(T(n)=aT(n/b)+\Theta(n^d)\).
  2. Identify \(a\), \(b\), and \(d\).
  3. Compute \(b^d\).
  4. Compare \(a\) with \(b^d\).
  5. Apply the matching case and simplify the result.

The theorem gives an asymptotic bound, not an exact closed form. That is usually what we want for algorithm analysis.

Example: Binary Search

Binary search has recurrence

\[ T(n)=T(n/2)+1. \]

Here \(a=1\), \(b=2\), and \(f(n)=1=\Theta(n^0)\), so \(d=0\). Since \(b^d=2^0=1\), we have \(a=b^d\). The second case gives

\[ T(n)=\Theta(n^0\log n)=\Theta(\log n). \]

Example: Merge Sort

Merge sort has recurrence

\[ T(n)=2T(n/2)+n. \]

Here \(a=2\), \(b=2\), and \(d=1\). Since \(b^d=2^1=2\), we have \(a=b^d\). The second case gives

\[ T(n)=\Theta(n\log n). \]

This matches the recursion-tree explanation: each level costs \(\Theta(n)\), and there are \(\Theta(\log n)\) levels.

Example: Root-Dominated Work

Consider

\[ T(n)=2T(n/2)+n^2. \]

Here \(a=2\), \(b=2\), and \(d=2\). Since \(b^d=2^2=4\), we have \(a<b^d\). The first case gives

\[ T(n)=\Theta(n^2). \]

The local work at the root is so large that the lower levels form a decreasing geometric series.

Example: Leaf-Dominated Work

Consider

\[ T(n)=7T(n/2)+n^2. \]

Here \(a=7\), \(b=2\), and \(d=2\). Since \(b^d=4\), we have \(a>b^d\). The third case gives

\[ T(n)=\Theta(n^{\log_2 7}). \]

Since \(\log_2 7\) is about \(2.807\), this grows a little faster than \(n^2\), but the exact exponent is \(\log_2 7\).

When It Does Not Apply

The Master Theorem is powerful, but it is not a universal recurrence solver. This simplified version does not apply directly to these recurrences:

Recurrence Why this version does not apply directly
\(T(n)=T(n-1)+n\) The subproblem size is \(n-1\), not \(n/b\).
\(T(n)=T(n/3)+T(2n/3)+n\) The recursive calls have different sizes.
\(T(n)=2T(n/2)+n\log n\) The local work is not \(\Theta(n^d)\) for a constant \(d\).
\(T(n)=T(n-1)+T(n-2)+1\) This is a linear recurrence with different previous terms, not a balanced divide-and-conquer recurrence.

When the theorem does not fit, try iteration, recursion trees, substitution, or a more advanced theorem if the course has introduced one.

Common Mistakes

  • Using \(b\) backward: If the subproblem size is \(2n/3\), then \(n/b=2n/3\), so \(b=3/2\), not \(2/3\).
  • Comparing \(a\) with \(d\): Compare \(a\) with \(b^d\), not with \(d\).
  • Dropping the logarithm in the equal case: When \(a=b^d\), the result is \(\Theta(n^d\log n)\).
  • Rounding an exponent into an equality: \(\Theta(n^{\log_2 7})\) is not the same as \(\Theta(n^{2.8})\), even though \(2.8\) is a useful approximation.
  • Forcing the theorem onto the wrong form: If the recurrence does not match the required shape, use another method.

Reading Comprehension Questions

  1. What do \(a\), \(b\), and \(f(n)\) represent in \(T(n)=aT(n/b)+f(n)\)?
  2. In the simplified version, what do we compare to choose the case?
  3. Why is merge sort the equal-case example?
  4. Why does the theorem not directly apply to \(T(n)=T(n-1)+n\)?
  1. Answer: \(a\) is the number of recursive calls, \(b\) is the shrink factor, and \(f(n)\) is the non-recursive work.
  2. Answer: Compare \(a\) with \(b^d\), where \(f(n)=\Theta(n^d)\).
  3. Answer: Merge sort has \(a=2\), \(b=2\), and \(d=1\), so \(a=b^d=2\).
  4. Answer: The recursive call is on \(n-1\), not on \(n/b\) for a constant \(b>1\).

In-Class Activities

  1. Parameter practice: For \(T(n)=3T(n/4)+n^2\), identify \(a\), \(b\), and \(d\).
  2. Case sorting: Sort \(2T(n/2)+n\), \(2T(n/2)+n^2\), and \(4T(n/2)+n\) into the three Master Theorem cases.
  3. Non-example discussion: Explain why \(T(n)=T(n/2)+T(n/4)+n\) does not fit this version.

Homework Problems

  1. Apply the theorem: Solve \(T(n)=4T(n/2)+n\).
  2. Equal case: Solve \(T(n)=9T(n/3)+n^2\).
  3. Root-dominated: Solve \(T(n)=3T(n/2)+n^3\).
  4. Careful shrink factor: Solve \(T(n)=3T(2n/3)+1\), identifying \(b\) correctly.
