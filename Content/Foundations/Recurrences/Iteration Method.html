Iteration Method

Overview

The iteration method solves a recurrence by repeatedly expanding the recursive term until a pattern appears. Once the pattern is clear, we choose the number of expansions needed to reach the base case and simplify.

This method is also called backward substitution. It is useful because it shows where the final formula comes from instead of treating a recurrence as something to memorize.

Basic Process

  1. Expand several times: Substitute the recurrence into itself for the smaller input.
  2. Keep algebra readable: Use forms such as \(2^3\) and \(n/2^3\) when they reveal the pattern.
  3. Generalize after \(k\) expansions: Write the expression after an arbitrary number of recursive steps.
  4. Find the base-case value of \(k\): Solve for the \(k\) that turns the remaining recursive term into the base case.
  5. Simplify: Substitute that \(k\) back into the expression and simplify the result.

The hardest part is usually not the recurrence idea. It is the algebra. Write enough intermediate steps that you can see which \(n\), \(n/2\), or \(n/4\) belongs in each term.

Example: Decrease by One

Start with a recurrence that looks like a recursive loop:

\[ T(n)=T(n-1)+3,\quad T(1)=2. \]

Expand the recurrence several times.

\[ \begin{aligned} T(n) &= T(n-1)+3\\ &= T(n-2)+3+3\\ &= T(n-3)+3+3+3\\ &= T(n-k)+3k. \end{aligned} \]

The base case occurs when \(n-k=1\), so \(k=n-1\). Substitute that value:

\[ T(n)=T(1)+3(n-1)=2+3n-3=3n-1. \]

The exact solution is \(3n-1\), so the growth rate is \(\Theta(n)\).

Example: Halving the Input

Binary search leads to a recurrence like this:

\[ T(n)=T(n/2)+1,\quad T(1)=1. \]

Each expansion halves the input and adds one more unit of local work.

\[ \begin{aligned} T(n) &= T(n/2)+1\\ &= T(n/4)+1+1\\ &= T(n/8)+1+1+1\\ &= T(n/2^k)+k. \end{aligned} \]

The base case occurs when \(n/2^k=1\). That means \(n=2^k\), so \(k=\log_2 n\). Therefore,

\[ T(n)=T(1)+\log_2 n=1+\log_2 n. \]

This gives the familiar \(\Theta(\log n)\) running time for binary search.

Example: Divide and Conquer

Now consider a recurrence with two half-size recursive calls and linear local work:

\[ T(n)=2T(n/2)+n,\quad T(1)=1. \]

Be careful to distribute the outer factor of 2 across the next expansion.

\[ \begin{aligned} T(n) &= 2T(n/2)+n\\ &= 2(2T(n/4)+n/2)+n\\ &= 2^2T(n/4)+n+n\\ &= 2^2(2T(n/8)+n/4)+2n\\ &= 2^3T(n/8)+3n\\ &= 2^kT(n/2^k)+kn. \end{aligned} \]

Again \(n/2^k=1\), so \(k=\log_2 n\). Substitute:

\[ T(n)=2^{\log_2 n}T(1)+n\log_2 n=n+n\log_2 n. \]

The tight asymptotic bound is \(\Theta(n\log n)\).

Example: Geometric Work

Some expansions create a geometric series. For example:

\[ T(n)=2T(n/2)+n^2,\quad T(1)=1. \]

Expanding gives

\[ \begin{aligned} T(n) &= 2T(n/2)+n^2\\ &= 2^2T(n/4)+2(n/2)^2+n^2\\ &= 2^2T(n/4)+\frac{n^2}{2}+n^2\\ &= 2^3T(n/8)+\frac{n^2}{4}+\frac{n^2}{2}+n^2\\ &= 2^kT(n/2^k)+\sum_{i=0}^{k-1}\frac{n^2}{2^i}. \end{aligned} \]

With \(k=\log_2 n\), the first term becomes \(nT(1)=n\). The summation is bounded above by \(2n^2\), so the total is \(\Theta(n^2)\). For asymptotic analysis, recognizing the geometric series is often enough.

When Iteration Helps

The iteration method works best when repeated expansion produces a clear pattern. It is usually a good first attempt for these shapes:

  • One recursive call, such as \(T(n)=T(n-1)+f(n)\) or \(T(n)=T(n/2)+f(n)\).
  • Balanced divide-and-conquer recurrences, such as \(T(n)=aT(n/b)+f(n)\).
  • Recurrences where the non-recursive work forms a familiar sum or geometric series.

It is less pleasant when a recurrence has many different recursive terms, such as \(T(n)=T(n-1)+T(n/3)+T(\sqrt n)+n\). In those cases, a different method or a looser bound may be more appropriate.

Common Mistakes

  • Substituting \(n\) where the smaller input belongs: When expanding \(T(n/2)\), the local work must be computed for input size \(n/2\).
  • Stopping after too few expansions: Two lines may suggest a false pattern. Expand enough to check it.
  • Choosing the wrong base-case equation: For \(T(n/2^k)\), solve \(n/2^k=1\). For \(T(n-k)\), solve \(n-k=1\).
  • Losing constants inside powers: \((n/2)^2=n^2/4\), not \(n^2/2\).
  • Forgetting the base-case term: The remaining \(T(1)\) still contributes a value, even if it does not change the asymptotic bound.

Reading Comprehension Questions

  1. What does the iteration method do to a recurrence?
  2. For \(T(n)=T(n/2)+1\), why is \(k=\log_2 n\)?
  3. In \(T(n)=2T(n/2)+n\), why does each level contribute \(n\) work?
  4. Why is it dangerous to infer a pattern from only one or two expansions?
  1. Answer: It repeatedly substitutes the recurrence into itself until the expression reaches a base case.
  2. Answer: After \(k\) halvings, the input is \(n/2^k\). Setting \(n/2^k=1\) gives \(n=2^k\), so \(k=\log_2 n\).
  3. Answer: At the next level there are twice as many subproblems, but each has half the input size, so \(2\cdot(n/2)=n\).
  4. Answer: Early lines can accidentally match several different patterns; more expansions help verify the general form.

In-Class Activities

  1. Pattern check: Expand \(T(n)=T(n-3)+2\) until you can write the expression after \(k\) steps.
  2. Halving practice: Use iteration to solve \(T(n)=T(n/2)+4\), \(T(1)=3\).
  3. Algebra audit: Compare two attempted expansions of \(T(n)=2T(n/2)+n^2\) and identify the first incorrect line.

Homework Problems

  1. Decrease by one: Solve \(A(n)=A(n-1)+2\), \(A(1)=2\), using iteration.
  2. Binary search variant: Solve \(B(n)=B(n/2)+5\), \(B(1)=1\), and give a tight asymptotic bound.
  3. Balanced split: Use iteration to solve \(C(n)=2C(n/2)+3n\), \(C(1)=1\), assuming \(n\) is a power of 2.
  4. Geometric levels: Use iteration to give a tight bound for \(D(n)=4D(n/2)+n\), \(D(1)=1\).
