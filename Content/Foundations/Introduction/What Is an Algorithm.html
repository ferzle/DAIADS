What Is an Algorithm?

Overview

An algorithm is a precise procedure for solving a problem. It takes an input, follows a finite sequence of well-defined steps, and produces an output. Algorithms are the ideas behind programs: a program is written in a particular language, but the algorithm is the method the program is carrying out.

For example, the instruction "sort these exam scores" describes a problem. Selection sort, insertion sort, merge sort, and quicksort are different algorithms for solving that problem. A Java method, a C++ function, and a Python function could all implement the same sorting algorithm.

Parts of an Algorithm

A useful algorithm description should make several things clear before anyone starts coding.

  • Input: What information is provided to the algorithm?
  • Output: What result should be produced?
  • Steps: What exact actions are performed, and in what order?
  • Stopping condition: How do we know the algorithm eventually finishes?
  • Correctness: Why should the output satisfy the problem requirements?
  • Cost: How much time and memory does the algorithm use as the input grows?

These parts matter because computers do not understand intent. A vague instruction such as "look through the data carefully" is not enough. The algorithm must say what is checked, how progress is made, and when the work is done.

Worked Example: Maximum Value

Suppose the problem is: given a nonempty list of numbers, return the largest number in the list. One natural algorithm scans the list from left to right while remembering the largest value seen so far. 

findMaximum(A):
    largest = A[0]
    for each remaining value x in A:
        if x > largest:
            largest = x
    return largest

Trace the algorithm on the list [7, 2, 9, 4, 9, 1].

Value inspected Question asked Largest after the step
7 Start here 7
2 Is 2 greater than 7? 7
9 Is 9 greater than 7? 9
4 Is 4 greater than 9? 9
9 Is 9 greater than 9? 9
1 Is 1 greater than 9? 9

The key idea is an invariant: after each step, largest stores the largest value among the elements already inspected. When every element has been inspected, that invariant tells us that largest is the largest value in the whole list.

Problem, Algorithm, Program, Data

It helps to separate four related ideas.

  • Problem: A task specification, such as "find whether a name appears in this list."
  • Algorithm: A general method for solving the task, such as sequential search or binary search.
  • Program: A concrete implementation of the method in a programming language.
  • Data structure: The way the data is organized so the algorithm can use it effectively.

The same problem can have many algorithms, and the same algorithm can have many implementations. Data structures also affect what algorithms are possible. Binary search is fast on a sorted array because the array gives direct access to the middle element. The same idea does not work well on a plain linked list, where reaching the middle requires walking through nodes one at a time.

Evaluating Algorithms

In this course, an algorithm is not judged only by whether it eventually returns an answer. We also ask whether it returns the right answer, how much work it performs, and what assumptions it makes about the input.

  • Correctness: Does the algorithm solve every valid input, not just the example we tried?
  • Time use: How many basic operations are performed as the input size grows?
  • Space use: How much additional memory is needed beyond the input itself?
  • Simplicity: Is the method easy to understand, implement, test, and maintain?
  • Robustness: What happens on edge cases such as empty input, duplicate values, or invalid data?

Later foundation pages discuss input size, cost models, and asymptotic notation in more detail. The goal for now is to see that algorithm design is about choosing a method deliberately, not just writing code until an example works.

Common Algorithm Patterns

Many algorithms are built from a small set of recurring patterns. Recognizing these patterns makes new algorithms easier to learn.

  • Scan: Visit items one at a time, as in finding a maximum or counting matches.
  • Search: Look for a target value or a state that satisfies a condition.
  • Sort: Rearrange data into an order that makes later work easier.
  • Split and combine: Break a problem into smaller parts, solve them, then combine the results.
  • Remember previous work: Store partial answers so repeated subproblems are not solved again.
  • Build step by step: Make a sequence of choices while maintaining a useful property.

These patterns reappear throughout brute force, divide-and-conquer, dynamic programming, greedy algorithms, graph algorithms, and data structure operations.

Reading Comprehension Questions

  1. Algorithm versus program: Why is an algorithm not the same thing as a program?
  2. Trace: In the maximum-value algorithm, what does largest mean after three values have been inspected?
  3. Data structure connection: Why does binary search depend on the way the data is stored?
  4. Evaluation: Name two questions besides "does it run?" that we should ask about an algorithm.
  1. Answer: An algorithm is the general method. A program is one language-specific implementation of that method.
  2. Answer: It is the largest value among the first three inspected values, not necessarily the largest in the full list yet.
  3. Answer: Binary search needs fast access to the middle item and assumes the data is sorted. Some structures make that efficient; others do not.
  4. Answer: Examples include whether it is correct for all valid inputs, how much time it uses, how much memory it uses, and how it handles edge cases.

In-Class Activities

  1. Write exact steps: Describe an algorithm for deciding whether a list contains duplicate values. Trade descriptions with a partner and test whether the steps are precise enough to follow.
  2. Trace by hand: Trace findMaximum on [3, 3, -2, 10, 5]. Record the value of largest after each comparison.
  3. Compare methods: Give two different algorithms for finding a name in a list. What assumptions does each method need?

Homework Problems

  1. Minimum and maximum: Write pseudocode that returns both the smallest and largest values in a nonempty list. Count how many comparisons your algorithm performs on a list of length n.
  2. Edge cases: Choose a simple algorithm you have written before. List at least five inputs that would be important to test, including boundary cases.
  3. Problem versus algorithm: Pick a common computing task, such as spell checking or route planning. State the problem separately from one possible algorithm for solving it.
