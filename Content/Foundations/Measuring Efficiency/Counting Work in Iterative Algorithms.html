Counting Work in Iterative Algorithms

Overview

Counting work turns code into a function of the input size. The count does not need to be exact. It needs to be accurate enough to identify the dominant growth pattern.

This page focuses on iterative code: straight-line statements, conditionals, sequential loops, and nested loops. Recursive algorithms are handled later with recurrence relations.

Straight-Line Code

Code with no loops and no recursive calls usually performs a constant amount of work. 

int max2(int x, int y) {
    if (x >= y) {
        return x;
    } else {
        return y;
    }
}

This method does one comparison and one return. The exact count depends on how detailed the cost model is, but the count does not grow with the input size. It is constant work.

Single Loops

If a loop executes \(n\) times and each iteration does constant work, the total work grows in proportion to \(n\). 

int countZeros(int[] a) {
    int count = 0;
    for (int i = 0; i < a.length; i++) {
        if (a[i] == 0) {
            count++;
        }
    }
    return count;
}

A good abstract operation is the comparison a[i] == 0. It executes once for each array element, so it executes \(n\) times for an array of length \(n\).

Sequential Loops

Loops that happen one after another add their work. 

for (int i = 0; i < n; i++) {
    total += a[i];
}

for (int j = 0; j < n; j++) {
    total += b[j];
}

The first loop does about \(n\) units of repeated work. The second also does about \(n\). Together they do about \(2n\). The factor \(2\) matters for exact timing, but not for the growth-rate classification.

Nested Loops

Nested loops multiply when the inner loop runs completely for each outer-loop iteration. 

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        table[i][j] = a[i] + a[j];
    }
}

The inner statement executes \(n\) times for each value of i. Since there are \(n\) values of i, the statement executes \(n \cdot n = n^2\) times.

Do not memorize "nested loops mean \(n^2\)." Read the loop bounds. The following code has a nested loop but still does work proportional to \(n\), because the inner loop always runs exactly 6 times. 

for (int i = 0; i < n; i++) {
    for (int j = 0; j < 6; j++) {
        total += a[i];
    }
}

Dependent Loop Bounds

When an inner loop's bound depends on an outer-loop variable, write the count as a sum. 

for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= i; j++) {
        total++;
    }
}

The inner statement executes \(1\) time when \(i=1\), \(2\) times when \(i=2\), and so on. The total is

\[ 1 + 2 + 3 + \cdots + n = \frac{n(n+1)}{2}. \]

The exact formula is useful, but for growth-rate analysis the important part is the leading \(n^2\) term.

Function Calls Are Not Automatically Constant

A common analysis mistake is treating every method call as one operation. That is only valid when the called method really is constant time. 

for (int i = 0; i < a.length; i++) {
    if (contains(b, a[i])) {
        matches++;
    }
}

If contains(b, a[i]) scans an array b of length \(m\), then each call may take \(m\) work, and the loop may take about \(nm\) work. If b is a hash set with expected constant-time lookup, the same loop may grow like \(n\).

Simplifying Counts

Counting work often gives expressions like these:

Detailed count Dominant growth Why
\(7\) constant The count does not depend on \(n\).
\(5n + 2\) linear The \(n\) term dominates the constant.
\(3n^2 + 10n + 8\) quadratic The \(n^2\) term dominates for large \(n\).
\(2n + 6m\) depends on \(n\) and \(m\) There are two independent input-size variables.

The next page formalizes this simplification using asymptotic notation.

Analysis Checklist

  1. Identify the input-size variable or variables.
  2. Choose the case being analyzed: best, worst, average, or expected.
  3. Choose one or more operations that capture the repeated work.
  4. Count how many times those operations execute.
  5. Check method calls and data structure operations instead of assuming they are constant.
  6. Simplify the result to its growth rate.

Reading Comprehension Questions

  1. Why does a single loop that executes \(n\) times usually grow linearly?
  2. Why do two sequential \(n\)-iteration loops not usually grow quadratically?
  3. Why should dependent nested loop bounds often be written as a summation?
  4. Why is it dangerous to assume every method call takes constant time?
  1. Answer: The repeated work happens once per input element, so the count is proportional to \(n\).
  2. Answer: Their work adds to about \(2n\); one loop is not repeated inside the other.
  3. Answer: The number of inner-loop iterations changes as the outer-loop variable changes.
  4. Answer: The called method may itself contain loops, recursion, or data structure operations whose cost depends on input size.

In-Class Activities

  1. Loop count: Count the number of times the innermost statement executes in a triangular nested loop where \(j\) runs from \(0\) to \(i-1\).
  2. Sequential versus nested: Write one code fragment with two sequential loops and one with two nested loops. Explain the difference in work.
  3. Hidden cost: Give an example where replacing an array lookup with a search through a list changes the growth rate of a loop.

Homework Problems

  1. Analyze loops: For each of three loop fragments, count the repeated operation and simplify the count to its dominant growth term.
  2. Dependent bounds: Analyze a nested loop where the inner loop runs from \(i\) to \(n\). Write the summation and simplify the leading term.
  3. Function call cost: Suppose contains takes linear time in a list of length \(m\). Analyze a loop that calls contains once for each of \(n\) elements.
