Prim's Algorithm

Problem Solved

Prim's Algorithm solves the Minimum Spanning Tree problem.

Design and Strategy

Prim's Algorithm builds a minimum spanning tree one vertex at a time. It starts from any chosen vertex and repeatedly adds the lightest edge that connects a vertex already in the tree to a vertex outside the tree. This is a greedy rule: at each step we commit to the cheapest edge that expands the current partial tree without creating a cycle.

A convenient implementation keeps, for every vertex not yet in the tree, the weight of the cheapest edge that could attach it to the tree. A min-priority queue then tells us which vertex should be added next. Whenever a new vertex enters the tree, we examine its incident edges and update those best-known attachment costs.

High-Level Algorithm

  1. Choose a start vertex \(s\).
  2. Set \(key[s] = 0\), and for every other vertex \(v\), set \(key[v] = \infty\).
  3. Insert all vertices into a min-priority queue keyed by their current \(key\) values.
  4. While the queue is not empty:
    • Extract the vertex \(u\) with minimum key.
    • Add \(u\) to the growing tree.
    • For each edge \((u,v)\) with weight \(w\), if \(v\) is not yet in the tree and \(w < key[v]\), update \(key[v]\) and record \(u\) as the parent of \(v\).
  5. The parent edges form the minimum spanning tree.
Small Example

Suppose the graph has edges \((A,B,2), (A,C,3), (B,C,1), (B,D,4), (C,E,2), (D,E,3)\), and we start at \(A\).

  1. Start with tree \(\{A\}\).
  2. The cheapest edge leaving the tree is \((A,B)\) with weight \(2\), so add \(B\).
  3. Now the cheapest edge crossing the cut is \((B,C)\) with weight \(1\), so add \(C\).
  4. Next add \((C,E)\) with weight \(2\).
  5. Finally add \((E,D)\) with weight \(3\).

The resulting tree has total weight \(2 + 1 + 2 + 3 = 8\).

Implementation in Java, C++, Python

The versions below use an adjacency list and a min-priority queue. They return the total weight of the minimum spanning tree for a connected graph. 

import java.util.*;

class Edge {
  int to, w;
  Edge(int to, int w) { this.to = to; this.w = w; }
}

int primMST(List<Edge>[] graph) {
  int n = graph.length;
  boolean[] inTree = new boolean[n];
  int[] key = new int[n];
  Arrays.fill(key, Integer.MAX_VALUE);
  key[0] = 0;

  PriorityQueue<int[]> pq =
      new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
  pq.add(new int[]{0, 0}); // {weight, vertex}

  int total = 0;
  while (!pq.isEmpty()) {
    int[] cur = pq.poll();
    int w = cur[0], u = cur[1];
    if (inTree[u]) continue;
    inTree[u] = true;
    total += w;

    for (Edge e : graph[u]) {
      if (!inTree[e.to] && e.w < key[e.to]) {
        key[e.to] = e.w;
        pq.add(new int[]{e.w, e.to});
      }
    }
  }
  return total;
}

struct Edge {

int primMST(const vector<vector<Edge>>& graph) {
  int n = graph.size();
  vector<bool> inTree(n, false);
  vector<int> key(n, INT_MAX);
  priority_queue<pair<int,int>, vector<pair<int,int>>, greater<pair<int,int>>> pq;

  key[0] = 0;
  pq.push({0, 0}); // {weight, vertex}

  int total = 0;
  while (!pq.empty()) {
    auto [w, u] = pq.top();
    pq.pop();
    if (inTree[u]) continue;
    inTree[u] = true;
    total += w;

    for (const Edge& e : graph[u]) {
      if (!inTree[e.to] && e.w < key[e.to]) {
        key[e.to] = e.w;
        pq.push({e.w, e.to});
      }
    }
  }
  return total;
}
import heapq

def prim_mst(graph):
    n = len(graph)
    in_tree = [False] * n
    key = [float("inf")] * n
    key[0] = 0
    pq = [(0, 0)]  # (weight, vertex)

    total = 0
    while pq:
        w, u = heapq.heappop(pq)
        if in_tree[u]:
            continue
        in_tree[u] = True
        total += w

        for v, edge_w in graph[u]:
            if not in_tree[v] and edge_w < key[v]:
                key[v] = edge_w
                heapq.heappush(pq, (edge_w, v))

    return total

Time/Space Analysis

Time Complexity: With an adjacency list and binary heap, Prim's Algorithm runs in \(O(E \log V)\), where \(V\) is the number of vertices and \(E\) is the number of edges.

Space Complexity: The graph representation, key array, and priority queue require \(O(V + E)\) space.

Variations/Improvements

On dense graphs, an adjacency-matrix version runs in \(O(V^2)\) and can be simpler to implement. Prim's is often preferred when the graph is represented by adjacency lists and a priority queue is readily available. A closely related alternative is Kruskal's Algorithm.

Reading Comprehension Questions

  1. Safe Edge: Why is the lightest edge leaving the current tree always a safe edge to add?
  2. Key Values: What does the key value of a vertex represent while Prim's Algorithm is running?
  1. Answer: It is the minimum-weight edge crossing the cut between vertices already in the tree and vertices outside it, so adding it preserves the possibility of an optimal MST.
  2. Answer: It is the weight of the cheapest known edge that can currently attach that vertex to the growing tree.
