Kruskal's Algorithm

Problem Solved

Kruskal's Algorithm solves the Minimum Spanning Tree problem.

Design and Strategy

Kruskal's Algorithm builds a minimum spanning tree one edge at a time. It sorts all edges by weight and then scans them from lightest to heaviest. An edge is accepted exactly when it connects two previously separate components; otherwise it is skipped because it would create a cycle.

The key data structure is a disjoint-set union structure, also called union-find. It quickly answers whether two vertices are already in the same connected component and merges components when a new edge is added to the tree.

High-Level Algorithm

  1. Sort all graph edges into nondecreasing order by weight.
  2. Place each vertex in its own singleton set.
  3. Process edges in sorted order:
    • If an edge \((u,v)\) connects two different sets, add it to the tree and union those sets.
    • If \(u\) and \(v\) are already in the same set, skip the edge.
  4. Stop after selecting \(V-1\) edges.
Small Example

Suppose the graph has edges \((A,B,2), (A,C,3), (B,C,1), (B,D,4), (C,E,2), (D,E,3)\).

  1. Sort edges by weight: \((B,C,1), (A,B,2), (C,E,2), (A,C,3), (D,E,3), (B,D,4)\).
  2. Add \((B,C)\).
  3. Add \((A,B)\).
  4. Add \((C,E)\).
  5. Skip \((A,C)\) because \(A\) and \(C\) are already connected.
  6. Add \((D,E)\) and stop.

The chosen edges form an MST of total weight \(1 + 2 + 2 + 3 = 8\).

Implementation in Java, C++, Python

These implementations sort the edges, use union-find to detect cycles, and return the total weight of the minimum spanning tree. 

import java.util.*;

class Edge {
  int u, v, w;
  Edge(int u, int v, int w) { this.u = u; this.v = v; this.w = w; }
}

class DSU {
  int[] parent, rank;
  DSU(int n) {
    parent = new int[n];
    rank = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;
  }
  int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]);
    return parent[x];
  }
  boolean union(int a, int b) {
    int ra = find(a), rb = find(b);
    if (ra == rb) return false;
    if (rank[ra] < rank[rb]) parent[ra] = rb;
    else if (rank[ra] > rank[rb]) parent[rb] = ra;
    else {
      parent[rb] = ra;
      rank[ra]++;
    }
    return true;
  }
}

int kruskalMST(int n, List<Edge> edges) {
  edges.sort(Comparator.comparingInt(e -> e.w));
  DSU dsu = new DSU(n);
  int total = 0;

  for (Edge e : edges) {
    if (dsu.union(e.u, e.v)) {
      total += e.w;
    }
  }
  return total;
}
#include <algorithm>
#include <vector>
using namespace std;

struct Edge {
  int u, v, w;
};

struct DSU {
  vector<int> parent, rank;
  DSU(int n) : parent(n), rank(n, 0) {
    for (int i = 0; i < n; ++i) parent[i] = i;
  }
  int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]);
    return parent[x];
  }
  bool unite(int a, int b) {
    int ra = find(a), rb = find(b);
    if (ra == rb) return false;
    if (rank[ra] < rank[rb]) parent[ra] = rb;
    else if (rank[ra] > rank[rb]) parent[rb] = ra;
    else {
      parent[rb] = ra;
      rank[ra]++;
    }
    return true;
  }
};

int kruskalMST(int n, vector<Edge>& edges) {
  sort(edges.begin(), edges.end(),
       [](const Edge& a, const Edge& b) { return a.w < b.w; });

  DSU dsu(n);
  int total = 0;
  for (const Edge& e : edges) {
    if (dsu.unite(e.u, e.v)) {
      total += e.w;
    }
  }
  return total;
}
def kruskal_mst(n, edges):
    parent = list(range(n))
    rank = [0] * n

    def find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]

    def union(a, b):
        ra, rb = find(a), find(b)
        if ra == rb:
            return False
        if rank[ra] < rank[rb]:
            parent[ra] = rb
        elif rank[ra] > rank[rb]:
            parent[rb] = ra
        else:
            parent[rb] = ra
            rank[ra] += 1
        return True

    total = 0
    for u, v, w in sorted(edges, key=lambda e: e[2]):
        if union(u, v):
            total += w
    return total

Time/Space Analysis

Time Complexity: Sorting the edges costs \(O(E \log E)\), and the union-find operations are nearly constant amortized time, so the overall running time is dominated by the sort.

Space Complexity: The edge list and disjoint-set structure require \(O(V + E)\) space.

Variations/Improvements

Kruskal's Algorithm is especially natural when the graph is already given as an edge list. Union by rank together with path compression makes cycle detection very fast in practice. A closely related alternative is Prim's Algorithm, which grows the tree from a starting vertex.

Reading Comprehension Questions

  1. Edge Ordering: Why does Kruskal's Algorithm sort all edges before making any selections?
  2. Skipping Edges: Under what condition is an edge skipped during the scan of the sorted edge list?
  1. Answer: The greedy rule is to consider the lightest remaining edge first, so sorting ensures edges are checked in the correct order.
  2. Answer: An edge is skipped when its endpoints are already in the same connected component, because adding it would create a cycle.

In-Class Activities

  1. Sort and Select: Give students an edge list, have them sort it by weight, and then mark each edge as accepted or rejected step by step.
  2. Union-Find Snapshot: Ask students to record the disjoint sets after every accepted edge so they can visualize the connected components merging.

Homework Problems

  1. Manual Simulation: Run Kruskal's Algorithm by hand on a weighted graph and list the accepted and rejected edges in order.
  2. Ties in Weights: Explain why two different tie-breaking choices among equal-weight edges can still lead to minimum spanning trees of the same total weight.
